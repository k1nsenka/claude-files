# タスク集約スキル（Task Summary Skill）

## 概要

日報やメモからタスクを抽出・集約し、優先度付けされたTODOリストを作成するスキル。

---

## 入力

- 対象期間の日報ファイル群（例: `20250113-日報.md` ～ `20250119-日報.md`）
- 研究議論プレゼン（例: `20250119-研究議論.pptx` or `20250119-研究議論.pdf`）
- 既存のTODOファイル（あれば）

### pptx, pdfファイルからの情報抽出

研究議論のpptx,pdfファイルには以下の情報が含まれることが多い：

| スライド位置 | 典型的な内容 | 抽出すべき情報 |
|-------------|-------------|---------------|
| 冒頭 | アジェンダ・議題 | 議論予定のトピック |
| 中盤 | 進捗報告・結果 | 完了タスク、得られた知見 |
| 終盤 | 次のアクション・課題 | 新規タスク、宿題事項 |
| 最終 | まとめ・TODO | 決定事項、期限付きタスク |

**pptxから抽出するタスクの例：**
- 「次回までに○○を実施」→ 新規タスク
- 「□□は継続検討」→ 保留タスク
- 「△△は完了」→ 完了タスク

## 出力

- 整理されたTODOリスト
- タスクのステータスサマリー

---

## 処理手順

### Step 1: タスク抽出

#### 日報（.md）からの抽出

以下のパターンを検出：

```
検出パターン:
- [ ] タスク内容（未完了）
- [x] タスク内容（完了）
- [-] タスク内容（キャンセル）
- [>] タスク内容（延期）
```

#### 研究議論pptx（.pptx）からの抽出

1. **スライドのテキストを順次読み取る**
2. **以下のキーワードを含むスライドに注目：**
   - 「TODO」「タスク」「宿題」「アクション」
   - 「次回まで」「来週まで」「○日まで」
   - 「検討事項」「課題」「残件」
3. **箇条書き項目をタスクとして抽出**

```
pptxでの典型的なタスク記述例：
• ○○の実験を追加で実施
• △△について文献調査
• □□のコードをリファクタリング
→ これらをタスクとして抽出
```

### Step 2: タスク分類

抽出したタスクを以下の観点で分類：

| 分類軸 | 分類項目 |
|-------|---------|
| ステータス | 未完了 / 完了 / 延期 / キャンセル |
| 優先度 | 高🔺 / 中🔸 / 低🔹 / 未設定 |
| 期限 | 期限超過 / 今日 / 今週 / 来週以降 / 未設定 |
| プロジェクト | タグまたはコンテキストから推定 |

### Step 3: 重複排除

- 同一タスクが複数日に記載されている場合は最新の状態を採用
- タスク文の類似度が高い場合は統合を検討

### Step 4: 優先度再評価

以下の基準で優先度を再評価・提案：

```
高優先度🔺の条件:
- 期限が今日または超過
- 他タスクの前提条件となっている
- 外部依存（他者待ち）の後続タスク

中優先度🔸の条件:
- 期限が今週内
- プロジェクトの主要マイルストーン

低優先度🔹の条件:
- 期限が来週以降または未設定
- nice-to-have なタスク
```

### Step 5: 出力生成

---

## 出力フォーマット

```markdown
# タスクサマリー（YYYY-MM-DD 作成）

## ステータス概要

| ステータス | 件数 |
|-----------|------|
| 未完了 | X件 |
| 完了（今週） | Y件 |
| 延期 | Z件 |

## 要対応（期限超過・今日）

- [ ] タスクA 📅 2025-01-19 🔺
  - 出典: [[20250118-日報]]
  - 備考: ○○が前提条件

## 今週のタスク

### 高優先度 🔺
- [ ] タスクB 📅 2025-01-22
  - 出典: [[20250117-日報]]

### 中優先度 🔸
- [ ] タスクC 📅 2025-01-24
  - 出典: [[20250115-日報]]

### 低優先度 🔹
- [ ] タスクD
  - 出典: [[20250116-日報]]

## 来週以降

- [ ] タスクE 📅 2025-01-30
- [ ] タスクF（期限未設定）

## 今週完了したタスク

- [x] タスクG ✅ 2025-01-18
- [x] タスクH ✅ 2025-01-17

## ⏸️ 延期・保留中

- [>] タスクI（理由: ○○待ち）
  - 元期限: 2025-01-15
  - 再開条件: △△が完了次第

## 💡 推奨アクション

1. **タスクA**を最優先で対応（期限超過）
2. **タスクB**は明日中に着手推奨
3. **タスクI**の再開条件を確認
```

---

## 使用例

### プロンプト例1: 週次タスク集約

```
今週の日報（20250113-日報.md ～ 20250119-日報.md）と
研究議論pptx（20250117-研究議論.pptx）から
タスクを抽出してサマリーを作成してください。
```

### プロンプト例2: プロジェクト別集約

```
#project/thesis タグが付いた日報から
修論関連のタスクのみを抽出してください。
```

### プロンプト例3: 期限超過タスクの確認

```
今日時点で期限が超過しているタスクを
すべての日報から抽出してください。
```

### プロンプト例4: pptxからのタスク抽出

```
20250119-研究議論.pptxの内容を確認し、
議論で決まったタスクや宿題事項を抽出してください。
```

---

## 注意事項

1. **コンテキスト保持**: タスクの出典（どの日報から抽出したか）を必ず記録
2. **備考の転記**: タスクに付随するメモや条件があれば合わせて抽出
3. **リンク維持**: `[[メモ名]]` 形式のリンクは保持
4. **定期実行推奨**: 週1回（週末または週初め）の実行を推奨
