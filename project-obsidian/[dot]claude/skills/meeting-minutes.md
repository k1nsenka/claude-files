# 会議議事録作成スキル（Meeting Minutes Skill）

## 概要

Zoomレコーディングのトランスクリプト（.vtt）と関連する会議資料（.md, .pdf, .pptx）を解析し、構造化された議事録を作成するスキル。

**優先度**: vttファイル（実際の発言内容） > 会議資料（事前準備・補足）

---

## 入力ファイル

### 必須

| ファイル | 命名パターン | 説明 |
|---------|-------------|------|
| Zoomトランスクリプト | `GMT{YYYYMMDD}-{HHMMSS}_Recording_transcript.vtt` | 会議の全発言記録 |

### オプション（同日付のファイルを自動検索）

| ファイル | 命名パターン | 説明 |
|---------|-------------|------|
| 研究議論メモ | `YYYYMMDD-研究議論*.md` | 事前準備メモ |
| 研究議論プレゼン | `YYYYMMDD*研究議論*.pdf` | 発表資料 |
| 日報 | `YYYYMMDD-日報.md` | 当日の作業記録 |

### 日付の抽出

vttファイル名から日付を抽出：
```
GMT20260121-005948_Recording_transcript.vtt
    ^^^^^^^^
    YYYYMMDD → 2026年01月21日
```

---

## 出力

- 構造化された議事録（.md）
- 次回会議までのTODOリスト

---

## 処理手順

### Step 1: vttファイルの解析

#### vttフォーマット構造

```
WEBVTT

1
00:00:11.870 --> 00:00:13.050
NEMOTO TATSUAKI(根本　龍昌): お疲れ様です。

2
00:00:13.780 --> 00:00:15.479
NEMOTO TATSUAKI(根本　龍昌): 少々お待ちください。
```

#### 抽出する情報

| 要素 | 抽出方法 |
|-----|---------|
| 発言者 | `名前(日本語名):` の形式から抽出 |
| タイムスタンプ | `HH:MM:SS.mmm --> HH:MM:SS.mmm` から開始時刻を取得 |
| 発言内容 | コロン以降のテキスト |

#### 発言者の正規化

```
入力例:
- NEMOTO TATSUAKI(根本　龍昌)
- [NEC] Kei Takemura
- SAKUMA KEITA(佐久間　啓太)

正規化後:
- 根本龍昌
- 竹村圭（NEC）
- 佐久間啓太
```

### Step 2: 会議内容の構造化

#### トピック分割

以下のキーワードでトピックの切り替わりを検出：

```
トピック開始の手がかり:
- 「では」「じゃあ」+ 新しい話題
- 「次に」「続いて」
- 「○○について」「○○の話」
- 「アジェンダ」「コンテンツ」
- タイムスタンプの大きなギャップ（5分以上）
```

#### 重要発言の検出

以下のパターンを重要発言として検出：

| パターン | 検出キーワード |
|---------|---------------|
| 決定事項 | 「決まりました」「にしましょう」「ことになりました」 |
| TODO | 「やります」「します」「しておきます」「次回までに」「宿題」 |
| 質問 | 「？」「どうですか」「いかがですか」「どう思います」 |
| 提案 | 「してはどうか」「した方がいい」「すべき」「ありかも」 |
| 懸念・課題 | 「問題」「課題」「難しい」「気になる」「おかしくね」 |
| 合意 | 「同感」「確かに」「そうですね」「分かりました」 |

### Step 3: TODO抽出（最重要）

vttから以下のパターンでTODOを抽出：

```
TODOの検出パターン:
1. 明示的TODO
   - 「○○をやる」「○○します」「○○しておきます」
   - 「次回までに○○」「来週○○」
   - 「宿題として○○」

2. 指示・依頼
   - 「○○してください」「○○してもらえますか」
   - 「○○した方がいい」「○○すべき」

3. 確認事項
   - 「確認しておきます」「調べておきます」
   - 「検討します」「考えておきます」

4. 文脈からの推測
   - 議論の結論として出たアクション
   - 次回への持ち越し事項
```

#### TODO抽出の優先度

| 優先度 | 条件 |
|-------|------|
| 🔺高 | 明確な期限言及（「明日」「今週中」「次回まで」） |
| 🔸中 | 担当者が明確 / 具体的なタスク |
| 🔹低 | 「検討」「考える」など曖昧なもの |

### Step 4: 関連ファイルとの照合

#### pptx/pdf/mdファイルからの補足

1. **アジェンダの確認**: pptx/pdfの目次スライドと実際の議論を照合
2. **図表の参照**: 議論で言及された図表をpptx/pdfから特定
3. **事前準備との差異**: 予定していた議題と実際の議論の差分を記録

### Step 5: 議事録生成

## 使用例

### プロンプト例1: 基本的な議事録作成

```
GMT20260121-005948_Recording_transcript.vtt を解析して
議事録を作成してください。
同日の研究議論ファイル（20260121-研究議論.pdfなど）があれば
それも参照してください。
```

### プロンプト例2: TODO重点抽出

```
GMT20260121-005948_Recording_transcript.vtt から
次回会議までのTODOを優先度付きで抽出してください。
担当者と期限を明確にしてください。
```

### プロンプト例3: 特定トピックの詳細

```
GMT20260121-005948_Recording_transcript.vtt の
「貢献度推定技術」に関する議論部分を詳しくまとめてください。
```

### プロンプト例4: 複数会議の比較

```
先週と今週のvttファイルを比較し、
進捗状況と新たに発生したTODOをまとめてください。
```

## 注意事項

### vtt解析時の留意点

1. **音声認識の誤り**: 固有名詞や専門用語は誤認識されやすい
   - 「マイルストーン」→「ワイルストーン」など
   - 文脈から推測して補正

2. **発言の断片化**: 一文が複数セグメントに分割されることがある
   - 連続するセグメントは文脈で繋げる

3. **フィラーの処理**: 「えっと」「まあ」などは要約時に省略

4. **オーバーラップ**: 複数人が同時に話している場合の処理

### 優先度の原則

```
vttファイル（実際の発言） >>>>>>> 会議資料（事前準備）

理由:
- 実際の議論で決まったことが最重要
- 事前資料と異なる結論になることがある
- アドホックな決定事項はvttにしかない
```
