# テクニカルライティングスキル（Technical Writing Skill）

## 概要

論文、レポート、技術文書を明確で論理的な構成で執筆するためのスキル。藤田肇氏が考案した「黄金フォーマット」に基づいた文書作成を支援。

---

## 黄金フォーマット

### なぜテクニカルライティングが重要か

適切なテクニカルライティングは**思考整理**そのもの。文章は映像に比べて情報量が少ないからこそ、**思考の本質を浮彫にする**必要がある。

適切なテクニカルライティングで明確にすべき2つの要素：

| 要素 | 定義 | 説明 |
|-----|------|------|
| **オリジナリティ** | 先行技術との差分 | 何が新しいのか |
| **インパクト** | 差分の大きさ | それがどれほど重要か |

---

### 4つの領域

黄金フォーマットは2×2の4つのボックスで構成される：

```
┌─────────────────────────────────────────────────────────────────┐
│                                                                 │
│   領域1：背景・前提           │   領域2：課題・仮説             │
│   ─────────────────           │   ─────────────────             │
│   ・それまでの状態            │   ・現在直面している問題や課題   │
│   ・今まで進行していたもの    │   ・課題と捉える理由             │
│   ・前提条件                  │   ・課題に対する仮説             │
│                               │                                 │
├───────────────────────────────┼─────────────────────────────────┤
│                               │                                 │
│   領域3：手段・行動           │   領域4：効果・結論             │
│   ─────────────────           │   ─────────────────             │
│   ・どう問題を解決するか      │   ・得られる結果から何が言えるか │
│   ・その手段を選んだ理由      │   ・なぜそれが言えるのか         │
│   ・アプローチの意味や意義    │   ・どのような変化が起こるか     │
│                               │                                 │
└─────────────────────────────────────────────────────────────────┘
```

---

### 領域間の関係性（最重要）

> **「ボックスの内容よりも、ボックス同士の関係が大切」** — 藤田肇

```
        領域1                              領域2
    ┌──────────────┐    【逆説関係】    ┌──────────────┐
    │   背景・前提  │ ←───相反する───→ │  課題・仮説   │
    └──────────────┘                    └──────────────┘
           │                                   │
           │                                   │
      【差分関係】                        【裏返し関係】
      現状との差分                       問題⇔解決
      のみを記載                         の裏返し
           │                                   │
           ↓                                   ↓
    ┌──────────────┐    【順接関係】    ┌──────────────┐
    │  手段・行動   │ ───原因→結果───→ │  効果・結論   │
    └──────────────┘                    └──────────────┘
        領域3                              領域4
```

#### 関係性の詳細

| 関係 | 領域 | 説明 | チェックポイント |
|-----|------|------|-----------------|
| **逆説関係** | 1↔2 | 現状と課題が相反し合う | 「しかし」「ところが」で繋がるか |
| **差分関係** | 1→3 | 手段は現状との差分のみ | 背景にない新しい要素だけを書いているか |
| **順接関係** | 3→4 | 原因と結果の関係 | 「したがって」「その結果」で繋がるか |
| **裏返し関係** | 2↔4 | 問題と解決が裏返し | 課題が解決された状態になっているか |

### 具体例：ラジオ搭載自転車

```
┌─────────────────────────────────────────────────────────────────┐
│                                                                 │
│   領域1：背景・前提                 │   領域2：課題・仮説       │
│   ─────────────────                 │   ─────────────────       │
│   自転車は移動手段として            │   しかし、自転車走行中は   │
│   広く普及している。                │   ラジオを聴くことが       │
│   また、ラジオは娯楽として          │   できないという課題がある │
│   多くの人に利用されている。        │                           │
│                                     │  【逆説：普及⇔利用不可】  │
├───────────────────────────────────────────────────────────────────┤
│                                     │                           │
│   領域3：手段・行動                 │   領域4：効果・結論       │
│   ─────────────────                 │   ─────────────────       │
│   そこで、自転車のハンドル部分に    │   これにより、自転車走行中 │
│   ラジオを搭載する。                │   でもラジオを楽しめる     │
│                                     │   ようになる。             │
│  【差分：ラジオ搭載という新要素】   │  【裏返し：課題の解決】    │
│                                     │  【順接：搭載→楽しめる】  │
└─────────────────────────────────────────────────────────────────┘
```

### ダメなパターン

> **「ダメ資料は単なるエッセイに過ぎない」** — 藤田肇

#### よくある失敗例

```
┌─────────────────────────────────────────────────────────────────┐
│                                                                 │
│   領域1：背景・前提                 │   領域2：課題・仮説       │
│   ─────────────────                 │   ─────────────────       │
│   ××についてやりました            │   △△が気になりました      │
│   （具体性がない）                  │   （課題が不明確）         │
│                                     │   （領域1と相反していない）│
├───────────────────────────────────────────────────────────────────┤
│                                     │                           │
│   領域3：手段・行動                 │   領域4：効果・結論       │
│   ─────────────────                 │   ─────────────────       │
│   □□を試してみました              │   ◇◇という結果でした      │
│   （なぜその手段を選んだか不明）    │   （課題との対応が不明）   │
│   （領域1との差分が不明確）         │   （領域3との因果が不明）  │
└─────────────────────────────────────────────────────────────────┘
```

#### 「読書感想文の罠」に陥った人がやってしまうこと

| ❌ やってしまうこと | ✅ あるべき姿 |
|-------------------|-------------|
| 思ったままに書く | 構造化して書く |
| 自由な書式で書く | フォーマットに従う |
| 時系列で書く | 論理構造で書く |
| 感想を書く | 分析を書く |

### 黄金フォーマット チェックリスト

文書作成時に以下を確認：

#### 各領域のチェック

| 領域 | チェック項目 |
|-----|-------------|
| 領域1 | □ 前提条件が明確に記述されているか |
| 領域1 | □ 従来の状況が具体的か |
| 領域2 | □ 課題が明確に定義されているか |
| 領域2 | □ なぜそれが課題なのか理由があるか |
| 領域2 | □ 仮説が立てられているか |
| 領域3 | □ 解決手段が具体的か |
| 領域3 | □ その手段を選んだ理由があるか |
| 領域3 | □ アプローチの意義が説明されているか |
| 領域4 | □ 結果から言えることが明確か |
| 領域4 | □ なぜそう言えるか根拠があるか |
| 領域4 | □ どんな変化が起こるか記述されているか |

#### 領域間関係のチェック

| チェック | 確認内容 |
|---------|---------|
| □ 1↔2 | 背景と課題が「しかし」で繋がる相反関係か |
| □ 1→3 | 手段は背景にない新しい差分のみを記載しているか |
| □ 3→4 | 手段から効果への因果関係が「したがって」で繋がるか |
| □ 2↔4 | 課題と結論が裏返しの関係（課題解決）になっているか |

## 実践：思考整理のための記述

### 記述時の留意点

| 観点 | 留意事項 |
|-----|---------|
| 具体性 | 抽象的な表現を避け、具体的に書く |
| 差分 | 何が新しいのかを明確にする |
| 因果 | 原因と結果の関係を明示する |
| 対応 | 課題と解決の対応を取る |

### 日常での活用

黄金フォーマットは以下の場面で活用できる：

- 障害発生時の報告書
- 新機能の提案書
- 研究の進捗報告
- 問題解決のドキュメント
- 日報・週報

## 文書タイプ別テンプレート

### 1. 研究メモ（黄金フォーマット適用）

```markdown
---
title: [研究テーマ]
date: YYYY-MM-DD
tags: [type/research, topic/xxx]
---

# [研究テーマ]

## 要点（3行で言うと）

1. [課題]: ...
2. [手段]: ...
3. [効果]: ...

## 領域1：背景・前提

[それまでの状態、前提条件]
- 従来の状況
- 既存の手法・アプローチ
- 前提となる知識

## 領域2：課題・仮説

[領域1と相反する問題]（「しかし」で繋がるように）
- 直面している問題
- なぜそれが課題なのか
- 課題に対する仮説

## 領域3：手段・行動

[領域1との差分のみ]
- 問題解決のアプローチ
- その手段を選んだ理由
- アプローチの意義

## 領域4：効果・結論

[領域2の裏返し、領域3からの順接]
- 得られた結果
- なぜそう言えるのか
- どのような変化が起こるか

## 参考文献

- [[関連メモ]]
```

### 2. 障害報告書（黄金フォーマット適用）

```markdown
---
title: 障害報告 - [概要]
date: YYYY-MM-DD
tags: [type/incident, system/xxx]
---

# 障害報告：[概要]

## 領域1：背景・前提

- 対象システム: 
- 通常時の状態: 
- 前提となる構成: 

## 領域2：課題・仮説

- 発生した問題: 
- 影響範囲: 
- 原因の仮説: 

## 領域3：手段・行動

- 実施した対応: 
- その対応を選んだ理由: 
- 対応の意図: 

## 領域4：効果・結論

- 対応結果: 
- 復旧確認方法: 
- 再発防止策: 

## タイムライン

| 時刻 | 事象 |
|-----|------|
| HH:MM | 障害発生 |
| HH:MM | 検知 |
| HH:MM | 対応開始 |
| HH:MM | 復旧確認 |
```

### 3. 提案書（黄金フォーマット適用）

```markdown
---
title: 提案 - [タイトル]
date: YYYY-MM-DD
tags: [type/proposal, project/xxx]
---

# [提案タイトル]

## エグゼクティブサマリー

> [3文で要約：課題→手段→効果]

## 領域1：現状

- 現在の状況
- これまでの経緯
- 前提条件

## 領域2：課題

- 解決すべき問題
- なぜ問題なのか（定量的に）
- 問題の原因仮説

## 領域3：提案内容

- 解決策の概要
- なぜこの方法を選ぶか
- 実施計画

## 領域4：期待効果

- 得られる効果（定量的に）
- 効果が得られる理由
- 組織への影響

## 補足情報

- コスト見積もり
- リスクと対策
- スケジュール
```

## 補助フレームワーク

### PREP法

短い説明や口頭発表では、PREP法も有効：

```
Point（結論）   → 最も伝えたいこと
Reason（理由）  → なぜそう言えるのか
Example（具体例）→ 根拠となる事実・データ
Point（結論）   → 結論の再提示
```

**黄金フォーマットとの使い分け**:

| フレームワーク | 用途 |
|--------------|------|
| 黄金フォーマット | 論文、レポート、提案書など構造化された文書 |
| PREP法 | プレゼン、口頭説明、短いメモ |

## 文章作成のルール

### 1. 一文一義

```markdown
# 悪い例
今回の実験では精度が向上したが、計算コストも増加したため、
実用性の観点からは検討が必要である。

# 良い例
今回の実験では精度が向上した。
一方で、計算コストも増加した。
実用性の観点からはトレードオフの検討が必要である。
```

### 2. 主語と述語を近づける

```markdown
# 悪い例
モデルAは、事前学習にWikipediaとBookCorpusを使用し、
ファインチューニングにはタスク固有のデータを用いて、
最終的に85%の精度を達成した。

# 良い例
モデルAは最終的に85%の精度を達成した。
事前学習にはWikipediaとBookCorpusを使用した。
ファインチューニングにはタスク固有のデータを用いた。
```

### 3. 具体的な数値を使う

```markdown
# 曖昧（避ける）
性能が大幅に向上した。

# 具体的（推奨）
性能が15%向上した（78% → 93%）。
```

### 4. 接続詞の適切な使用

| 関係 | 接続詞 | 黄金フォーマットでの使用 |
|-----|--------|------------------------|
| 逆説 | しかし、ところが、一方で | 領域1→2 |
| 順接 | したがって、その結果、ゆえに | 領域3→4 |
| 並列 | また、さらに、加えて | 同一領域内 |
| 例示 | 例えば、具体的には | 根拠説明 |
