# 進捗報告作成スキル（Progress Report Skill）

## 概要

サーベイメモ、研究議論メモ、日報から情報を集約し、進捗報告会での発表内容を構成するスキル。

---

## 入力

- サーベイメモ（`YYYYMMDD-サーベイ-*.md`）
- 研究議論メモ（`YYYYMMDD-研究議論-*.md`）
- 研究議論プレゼン（`mtg-doc/YYYYMMDD-研究議論.pdf`）
- 日報（`YYYYMMDD-日報.md`）
- 発表時間の制約（例: 5分、10分、15分）
- 報告会の種類（ラボMTG、中間発表、学会発表など）

## 出力

- 会議の目標
- 発表用アウトライン
- 各セクションの話すべきポイント
- 想定Q&A

---

## 処理手順

### Step 1: 情報収集

対象期間のメモ・ファイルから以下を抽出：

| ファイル種別 | 抽出項目 |
|-------------|---------|
| サーベイ（.md） | 読んだ論文、キーポイント、自研究との関連 |
| 研究議論（.md） | 議論内容、得られた示唆、決定事項、宿題 |
| 研究議論（.pptx） | 発表内容、図表、議論結果、次のアクション |
| 日報（.md） | 実施した実験、得られた結果、発生した問題 |

#### pptx,pdfファイルからの情報抽出

研究議論pptx,pdfは以下の構造で情報を持つことが多い：

```
典型的なスライド構成:
1. タイトル・日付・参加者
2. 前回からの進捗
3. 今回の報告内容（図表含む）
4. 議論ポイント
5. 次のアクション・TODO
```

**pptx,pdfから抽出すべき情報：**
- **図表**: 再利用可能なグラフ・表・図
- **数値データ**: 実験結果、性能指標
- **議論の結論**: 決定事項、方針
- **TODO**: 次回までの宿題、期限

### Step 2: トピック分類

抽出した情報を以下のカテゴリに分類：

```
1. 進捗（何をやったか）
   - 実験・実装
   - サーベイ・調査
   - 執筆・資料作成

2. 結果（何がわかったか）
   - 実験結果
   - 新しい知見
   - 問題点の発見

3. 考察（どう解釈するか）
   - 結果の意味
   - 先行研究との比較
   - 仮説の検証状況

4. 次のアクション（何をするか）
   - 次の実験計画
   - 解決すべき課題
   - 相談事項
```

### Step 3: 時間配分

発表時間に応じた構成を決定：

#### 5分発表（ラボMTG向け）

```
構成:
1. 今週やったこと（1分）
2. 主要な結果・知見（2分）
3. 課題と次のアクション（1分）
4. 相談事項（1分）
```

#### 10分発表（進捗報告会向け）

```
構成:
1. 背景・目的のリマインド（1分）
2. 前回からの進捗概要（1分）
3. 詳細な結果報告（4分）
4. 考察・議論（2分）
5. 次の計画（1分）
6. 相談事項（1分）
```

#### 15分発表（中間発表向け）

```
構成:
1. 研究背景・動機（2分）
2. 関連研究（2分）
3. 提案手法（3分）
4. 実験設定（2分）
5. 結果と考察（4分）
6. 今後の予定（2分）
```

### Step 4: コンテンツ生成

各セクションについて以下を作成：

1. **箇条書きアウトライン**: 話す順序と要点
2. **キーメッセージ**: そのセクションで最も伝えたいこと
3. **補足情報**: 質問された場合に答える内容
4. **出典リンク**: 詳細を参照できるメモへのリンク

### Step 5: Q&A準備

議論メモや実験結果から想定される質問を抽出：

```
想定質問のカテゴリ:
- 手法の妥当性に関する質問
- 実験設定に関する質問
- 結果の解釈に関する質問
- 今後の方針に関する質問
- 先行研究との差異に関する質問
```

## 使用例

### プロンプト例1: 週次進捗報告

```
先週（01/13-01/19）のサーベイメモ、日報、
および20250117-研究議論.pptxから、
明日のラボMTG用の5分発表資料を作成してください。
```

### プロンプト例2: 特定トピックの報告

```
LLM推論に関する今月のサーベイメモから、
10分の進捗報告を作成してください。
実験結果の図は20250117-日報にあります。
```

### プロンプト例3: Q&A準備

```
来週の中間発表に向けて、
これまでの研究議論メモから想定Q&Aを作成してください。
```

### プロンプト例4: pptxからの報告作成

```
20250115-研究議論.pptxと20250119-研究議論.pptxの
内容を比較し、この2週間の進捗をまとめてください。
```

### プロンプト例5: 図表の再利用

```
過去の研究議論pptx（20250110-研究議論.pptx～20250119-研究議論.pptx）から
実験結果のグラフがあるスライドを探し、
進捗報告で再利用できる図表をリストアップしてください。
```

## Tips

### 効果的な進捗報告のポイント

0. **目的を明確化**: この議論では何を目的とするのかを明確に言う
1. **結論ファースト**: 最初に一番伝えたいことを言う
2. **具体的な数値**: 「改善した」ではなく「5%向上した」
3. **視覚的な資料**: 図表を効果的に活用
4. **次のアクションを明確に**: 何をいつまでにやるか
5. **相談事項を整理**: 議論したいポイントを明確に

### 時間管理

- 各セクションの時間を厳守
- 詳細は「質問があれば」に回す
- 練習して時間を計測
