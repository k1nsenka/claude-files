# セキュリティルール

## 機密ファイルの取り扱い

- `.env` ファイルや APIキー・トークン・認証情報を含むファイルの読み取り・変更は絶対に避ける
- 機密ファイルの編集が必要な場合は、ユーザに連絡して許可を得る

## ガイドライン

- 機密情報を絶対にコミットしない。シークレットは環境変数で扱う
- ログや出力に認証情報を含めない
- `pyproject.toml` に認証情報やトークン入り URL を含めない
- `.gitignore` に必須: `.env`, `.env.*`, `*.pem`, `*.key`, `secrets/`
