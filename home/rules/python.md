# Python ルール（uv + コーディングスタイル）

## 環境管理は uv のみ

- パッケージ追加は `uv add`（開発用は `uv add --dev`）、削除は `uv remove`
- スクリプト実行は必ず `uv run`（`uv run python`, `uv run pytest`, `uv run ruff` など）
- **禁止**: `pip install`、`python -m venv`、`python` の直接実行、グローバル環境へのインストール
- `pyproject.toml` と `uv.lock` は Git 管理する
- 作業開始時は `uv sync` で依存関係を同期する

## コーディングスタイル

- PEP 8 に従う。品質チェックは `uv run ruff check .` / `uv run ruff format .` / `uv run mypy src/`
- シンプルで読みやすいコード、適切な命名、一関数一責務
- エラーハンドリングを適切に実装し、コメントは必要な箇所にのみ付ける

## 設定値管理

- 設定可能なパラメータのハードコード禁止。設定ファイル/設定オブジェクトから取得する
- 設定の読み込み失敗をデフォルト値で握りつぶさない。明示的にエラーとして報告する
- 使用する設定値はログ出力し、どこで使われているか追跡可能にする

## 完全実装の原則

- 簡易実装・モック実装で完了とみなさない（「TODO」「簡易実装」コメントを残さない）
- 設計書と異なる実装が必要な場合は、事前にユーザに許可を取る
- LLM API 呼び出しが必要な機能は、プロンプト定義・レスポンス処理・エラーハンドリングを含めて実装する
