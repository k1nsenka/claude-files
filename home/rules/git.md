# Git ルール

## コミット前の確認

- git コミットを行う前に**必ずユーザに確認を取る**
- 変更内容の概要を説明し、コミットしてよいか確認を求める
- ユーザの承認なしにコミットを実行しない

## コミットメッセージ

- prefix を付ける: `feat` / `fix` / `refactor` / `docs` / `test` / `chore`
- prefix の後は日本語で「何をどのように変更したか」を具体的に書く
- 「更新」「作業完了」のような抽象的表現は禁止

良い例:

```
feat: ユーザー認証機能を追加
fix: ログイン時のトークン検証エラーを修正
refactor: データベース接続処理を共通化
```

## ブランチ運用

- 機能開発は `feature/`、バグ修正は `fix/` ブランチで行う
- main への直接 push は禁止
- GitHub 操作（PR・Issue の確認/作成）は gh コマンドを使用する
