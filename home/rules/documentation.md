# ドキュメントルール（開発プロジェクトの docs/ 配下）

## 設計書

- 新規開発時は `docs/` 以下に要件定義書と設計書（概略・機能・クラス構成）を作成する
- 既存ソフトウェアの修正時は既存の設計書を参照し、修正内容に応じて設計書も更新する
- 設計書を作成したら、コードを書く前にユーザーにチェックを依頼する

## ファイル命名規則

- 形式: `YYYY-MM-DD_内容を表す日本語名.md`（例: `2026-07-09_システム設計書.md`）
- 日付はシステムの現在時刻から自動取得する（タイムゾーン: Asia/Tokyo）
