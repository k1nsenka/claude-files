# 査読パネル（並列マルチレビュー + メタレビュー）

## 概要

論文1本に対して5つの異なる専門視点の査読を**並列**で実行し、最後にメタレビューで統合する。

| 査読者 | 視点 |
|--------|------|
| reviewer-a | 方法論・妥当性・再現性 |
| reviewer-b | 分析の深さ（なぜ効くか・失敗モード）"Analysis Lover" |
| reviewer-c | 産業応用・実用性・導入可能性 "Industry Pragmatist" |
| reviewer-d | 理論の厳密性 "Theoretical Stickler" |
| reviewer-e | 実験設計・比較の公平性・統計 "Empirical Hardliner" |

## 実行手順

### Step 1: 入力確認

- 論文ファイルのパスを特定する（PDF / Markdown / LaTeX）
- 引数や依頼文に「top-tier」が含まれるかを判定し、査読モードを決定する

### Step 2: 5査読者の並列実行

Agent ツールで reviewer-a 〜 reviewer-e の5エージェントを**1つのメッセージで同時に**起動する。
各エージェントへのプロンプトには以下を含める:

- 論文ファイルのパス
- 査読モード（標準 / top-tier）
- 「paper-review スキルの手順と出力形式に従い、自分の専門視点を最重要視して査読すること」
- 「最終出力として査読レポート全文を返すこと」

### Step 3: メタレビュー

5件の査読結果が揃ったら、meta-reviewer エージェントを起動し、5件の査読レポート全文をプロンプトに含めて渡す。
meta-reviewer は meta-review スキルの形式（合意点/不一致点、Key Issues、必須/推奨修正、最終判定、著者対応ガイダンス）で統合する。

### Step 4: 結果の提示

- メタレビュー全文をユーザーに提示する
- 各査読者の個別判定を一覧表（査読者/判定/最大の懸念）で添える
- 依頼があれば、結果を `YYYYMMDD-査読-[論文名].md` としてファイル保存する

## 注意事項

- 5エージェントは必ず並列（同一メッセージ内で複数 Agent 呼び出し）で起動し、逐次実行しない
- 個別査読の生データはメタレビューの根拠として保持し、ユーザーから求められたら提示する
