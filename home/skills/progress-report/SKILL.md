# 進捗報告作成スキル

サーベイメモ、研究議論メモ、日報から情報を集約し、進捗報告会での発表内容（アウトライン + 想定Q&A）を構成する。

## 入力

- サーベイメモ（`YYYYMMDD-サーベイ-*.md`）、研究議論メモ（`YYYYMMDD-研究議論-*.md`）
- 研究議論プレゼン（`YYYYMMDD-研究議論.pptx/.pdf`）、日報（`YYYYMMDD-日報.md`）
- 発表時間の制約（5分/10分/15分）と報告会の種類（ラボMTG、中間発表など）

## 処理手順

### Step 1: 情報収集

| ファイル種別 | 抽出項目 |
|-------------|---------|
| サーベイ | 読んだ論文、キーポイント、自研究との関連 |
| 研究議論メモ | 議論内容、得られた示唆、決定事項、宿題 |
| 研究議論 pptx/pdf | 図表、数値データ、議論の結論、次のアクション・TODO |
| 日報 | 実施した実験、得られた結果、発生した問題 |

### Step 2: トピック分類

1. **進捗**（何をやったか）: 実験・実装 / サーベイ / 執筆
2. **結果**（何がわかったか）: 実験結果 / 新しい知見 / 問題点の発見
3. **考察**（どう解釈するか）: 結果の意味 / 先行研究との比較 / 仮説の検証状況
4. **次のアクション**（何をするか）: 次の実験計画 / 課題 / 相談事項

### Step 3: 時間配分

- **5分（ラボMTG）**: 今週やったこと1分 → 主要な結果2分 → 課題と次のアクション1分 → 相談事項1分
- **10分（進捗報告会）**: 背景リマインド1分 → 進捗概要1分 → 詳細結果4分 → 考察2分 → 次の計画1分 → 相談1分
- **15分（中間発表）**: 背景・動機2分 → 関連研究2分 → 提案手法3分 → 実験設定2分 → 結果と考察4分 → 今後の予定2分

### Step 4: コンテンツ生成

各セクションについて: 箇条書きアウトライン / キーメッセージ / 補足情報（質問された場合の回答）/ 出典メモへのリンク

### Step 5: Q&A準備

議論メモや実験結果から想定質問を抽出: 手法の妥当性 / 実験設定 / 結果の解釈 / 今後の方針 / 先行研究との差異

## 出力フォーマット

```markdown
# 進捗報告（YYYY-MM-DD）

## メタ情報
- 報告会 / 発表時間 / 対象期間

## キーメッセージ（30秒で言うなら）
> 今週は○○を行い、△△という結果を得た。次は◇◇を検証する。

## 発表アウトライン
### 0. 今回の議論の目的
### 1..N 各セクション（時間付き）
**話すこと:**（箇条書き）
**キーメッセージ:**（1行）
**見せる図表:**（あれば）
**詳細参照:** [[出典メモ]]

## 想定Q&A
### Q1: ...
**想定回答:** ... **詳細参照:** [[メモ]]

## 参照メモ一覧
```

## 効果的な進捗報告のポイント

1. **目的を明確化**: この議論で何を得たいかを最初に言う
2. **結論ファースト**: 一番伝えたいことを最初に
3. **具体的な数値**: 「改善した」ではなく「5%向上した」
4. **次のアクションを明確に**: 何をいつまでにやるか
5. **相談事項を整理**: 議論したいポイントを明確に
