# リファクタリングワークフロー

## 手順

1. **事前確認**: `uv sync` で環境同期し、`uv run pytest` で現在のテスト状態を把握する
2. **現状分析と計画**: リファクタリング対象を特定し、計画をユーザーに報告する
3. **実施**: 下記チェックリストに従って作業する
4. **検証**: `uv run pytest -v` で全テスト成功を確認する
5. **報告**: 完了した項目と結果を報告する

## チェックリスト

### コード
- 未使用の関数・クラス・変数・コードの削除
- 重複コードの統合、構造の最適化
- TODO / FIXME / HACK / BUG コメントの処理
- `uv run ruff check --fix .` / `uv run ruff format .` / `uv run mypy src/`

### ドキュメント
- 不要ドキュメントの削除、既存ドキュメント・README の最新化
- 命名規則（`YYYY-MM-DD_日本語名.md`）への準拠確認

### テスト
- 不要テストの削除、カバレッジの確認
- `uv run pytest` 全テスト成功

## 重要なルール

- 機能変更を伴うリファクタリングは事前にユーザーに確認する
- テストが通らない状態で作業を完了としない
