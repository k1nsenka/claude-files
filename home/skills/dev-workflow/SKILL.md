# 開発ワークフロー（設計 → 承認 → 実装 → テスト）

## フェーズ1: 設計（「設計して」「要件定義して」の場合）

1. `docs/` に要件定義書と設計書（概略・機能・クラス構成）を作成する
   - 命名規則: `YYYY-MM-DD_内容を表す日本語名.md`
2. 作成したらユーザーに確認を依頼し、承認まで議論を続ける

**重要**: 設計フェーズでは絶対にコードを書かない。**ユーザーの承認なしに実装フェーズへ進まない。**

## フェーズ2: 実装（「実装して」「開発して」の場合）

0. `uv sync` で環境を同期する
1. 設計書を確認し、設計内容を踏襲して実装する
2. 設計書と異なる実装が必要になったら、事前にユーザに確認する
3. 簡易実装・モック実装は禁止。「TODO」「簡易実装」コメントを残したまま完了としない
4. `uv run pytest` を実行し、**すべてのテストが成功するまで修正する**

## 修正フロー（「修正して」の場合）

1. まず設計書を更新してユーザに確認し、合意できたらコードを修正する
2. 例外: 設計変更を伴わない軽微な修正は直接コード修正してよい

## 完了チェックリスト

- [ ] 設計書に準拠している（差異はユーザ承認済み）
- [ ] 完全な実装になっている（簡易実装でない）
- [ ] エラーハンドリングが適切
- [ ] `uv run pytest` 全パス
