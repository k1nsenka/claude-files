# タスク集約スキル

日報やメモからタスクを抽出・集約し、優先度付けされた TODO リストを作成する。週1回（週末または週初め）の実行を推奨。

## 入力

- 対象期間の日報ファイル群（例: `20260706-日報.md` 〜 `20260712-日報.md`）
- 研究議論プレゼン（`YYYYMMDD-研究議論.pptx/.pdf`）、既存の TODO ファイル（あれば）

## 処理手順

### Step 1: タスク抽出

**日報（.md）から**: `- [ ]`（未完了）/ `- [x]`（完了）/ `- [-]`（キャンセル）/ `- [>]`（延期）を検出。

**研究議論 pptx/pdf から**: スライドを順次読み、「TODO」「宿題」「アクション」「次回まで」「検討事項」「課題」「残件」を含むスライドの箇条書きをタスクとして抽出。
- 「次回までに○○を実施」→ 新規タスク / 「□□は継続検討」→ 保留タスク / 「△△は完了」→ 完了タスク

### Step 2: タスク分類

ステータス（未完了/完了/延期/キャンセル）× 優先度（🔺/🔸/🔹）× 期限（超過/今日/今週/来週以降）× プロジェクト（タグや文脈から推定）で分類。

### Step 3: 重複排除

- 同一タスクが複数日に記載されている場合は最新の状態を採用
- 類似度が高いタスクは統合を検討

### Step 4: 優先度再評価

- **🔺高**: 期限が今日または超過 / 他タスクの前提条件 / 外部依存の後続タスク
- **🔸中**: 期限が今週内 / 主要マイルストーン
- **🔹低**: 期限が来週以降または未設定 / nice-to-have

### Step 5: 出力生成

## 出力フォーマット

```markdown
# タスクサマリー（YYYY-MM-DD 作成）

## ステータス概要
| ステータス | 件数 |（未完了 / 完了（今週）/ 延期）

## 要対応（期限超過・今日）
- [ ] タスクA 📅 期限 🔺
  - 出典: [[日報]] / 備考: 前提条件など

## 今週のタスク
### 高優先度 🔺 / ### 中優先度 🔸 / ### 低優先度 🔹
- [ ] タスク 📅 期限
  - 出典: [[日報]]

## 来週以降

## 今週完了したタスク
- [x] タスク ✅ 完了日

## ⏸️ 延期・保留中
- [>] タスク（理由 / 元期限 / 再開条件）

## 💡 推奨アクション
1. 最優先タスクと着手推奨順
```

## 注意事項

1. **コンテキスト保持**: タスクの出典（どの日報から抽出したか）を必ず記録する
2. **備考の転記**: タスクに付随するメモや条件も合わせて抽出する
3. **リンク維持**: `[[メモ名]]` 形式のリンクは保持する
