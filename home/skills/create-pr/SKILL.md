# プルリクエスト作成ワークフロー

## 手順

1. **事前確認**: `gh auth status` / 現在のブランチ / `git status` と `git diff --stat main` で変更内容を確認
2. **テスト**: `uv run pytest` で全テストがパスすることを確認（Pythonプロジェクトの場合）
3. **コミット**: 未コミットの変更があれば、ユーザーに確認を取ってからコミット
4. **プッシュ**: `git push -u origin <ブランチ名>`
5. **PR作成**: **作成前に必ずユーザーに確認を取り**、承認後に `gh pr create` で作成
6. **確認**: `gh pr view` で作成結果を提示

## PRテンプレート

タイトル: `[prefix] 変更内容の要約`（例: `[feat] ユーザー認証機能の追加`）

本文:

```markdown
## 概要
この PR で行った変更の概要

## 変更内容
- 変更点1
- 変更点2

## テスト
- [ ] ユニットテスト追加/更新
- [ ] `uv run pytest` 全パス

## 関連Issue
closes #123
```

## 重要なルール

- PR 作成前に必ずユーザーの承認を得る
- タイトルと説明は具体的に記述する
- main ブランチへの直接プッシュは禁止
