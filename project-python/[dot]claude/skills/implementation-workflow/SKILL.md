# 実装ワークフロー

## 概要
このスキルは実装・修正フェーズでの作業手順を定義します。

## 事前準備（uv環境）

### Step 0: 環境確認
```bash
# 依存関係の同期
uv sync

# Pythonバージョン確認
uv run python --version
```

## 実装フェーズ（「開発して」「実装して」の場合）

### Step 1: 設計書の確認
設計内容を踏襲して開発を進める。

### Step 2: 設計との差異確認
設計書と異なる実装が必要な場合は事前にユーザに確認。

### Step 3: 実装
- 設計書通りの実装を行う
- 簡易実装・モック実装は禁止
- 「TODO」「簡易実装」等のコメントを残さない

### Step 4: テスト実行
```bash
uv run pytest
```
すべてのテストが成功するまで修正。

## 修正フェーズ（「修正して」の場合）

### Step 1: 設計書の更新
まず設計書を変更してユーザに確認。

### Step 2: ユーザー承認
問題ないことが合意できたらコードを修正。

### Step 3: 例外
設計変更なしの軽微な修正の場合は直接コード修正可能。

## パッケージ追加が必要な場合
```bash
# 本番用
uv add package-name

# 開発用
uv add --dev package-name
```

## 品質チェックリスト
- [ ] `uv sync` で環境同期済み
- [ ] 設計書に準拠しているか
- [ ] 完全な実装になっているか（簡易実装でないか）
- [ ] エラーハンドリングは適切か
- [ ] `uv run pytest` で全テストパス
- [ ] PEP 8に準拠しているか
