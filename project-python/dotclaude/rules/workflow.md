# ワークフロールール

## 基本ルール

### 指示への対応
- 要件や指示に従って作業を進める
- 作業の進捗や問題が発生した場合は適宜報告

### 自律的な問題解決
- エラーや問題が発生したら、自律的に問題分析と解決案を提示
- 複数のアプローチがある場合は、推奨案を明示
- ソースコード外の問題である可能性がある場合は、指示者に報告

### 既存コードの尊重
- 既存のコードスタイルやパターンがある場合には、それに従う
- 大幅な変更が必要な場合は理由を説明

### 連続失敗時の対応
- 2回以上連続でテストを失敗した時は、現在の状況を整理して指示者に報告
- 同じことを連続で行うのではなく、問題の解決策を提案

## Python環境管理（uv使用）

### 作業開始前
```bash
# 依存関係の同期（仮想環境も自動作成）
uv sync

# Pythonバージョン確認
uv run python --version
```

### パッケージ追加が必要な場合
```bash
# 本番用
uv add package-name

# 開発用
uv add --dev package-name
```

### スクリプト実行
```bash
# 必ず uv run を使用
uv run python script.py
uv run pytest
```

### 禁止事項
- `pip install` の使用
- `python` の直接実行（`uv run python` を使用）
- グローバル環境へのインストール
