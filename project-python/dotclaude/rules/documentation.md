# ドキュメントルール

## 設計書の構成
新規開発時は `docs` ディレクトリ以下に以下の内容を含む設計書を作成:
- 要件定義書
- 設計書（概略・機能・クラス構成）

## 既存ソフトウェアの修正時
- 既存の設計書を参照してソフトウェアを開発
- 修正内容に応じて設計書も更新

## ユーザー確認
- 設計書を作成したら、コードを作成する前にユーザーに設計書のチェックを依頼

## ファイル命名規則

### 基本ルール
- docsディレクトリ内のファイル名は日本語で作成し、作成日付を含める

### 形式
`YYYY-MM-DD_内容を表す日本語名.md`

### 例
- `2025-01-10_システム設計書.md`
- `2025-01-15_実装フロー分析.md`

### 構成要素
- **日付部分**: YYYY-MM-DD形式（ISO 8601準拠）
- **区切り文字**: アンダースコア（_）で日付と内容を区切る
- **内容部分**: 主要な内容を表す日本語
- **拡張子**: `.md`

### 日付の意味
- ドキュメント作成日または最終更新日を表す

### 日付取得方法
- システムの現在時刻から日付を自動取得（手動入力ではない）
- タイムゾーン: 日本時間（Asia/Tokyo）
