# リファクタリングワークフロー

## 概要
このスキルはリファクタリング作業の手順を定義します。

## 事前準備
```bash
# 環境同期
uv sync

# 現在のテスト状態を確認
uv run pytest
```

## チェックリスト

### 1. ソースコードの整理
- [ ] 無駄なソースコードの削除
- [ ] 使用されていない関数・クラス・変数の削除
- [ ] 重複コードの統合
- [ ] コード構造の最適化
- [ ] TODO、FIXME、HACK、BUGコメントの処理

### 2. コード品質チェック
```bash
# リンター
uv run ruff check .
uv run ruff check --fix .

# フォーマット
uv run ruff format .

# 型チェック（mypyがある場合）
uv run mypy src/
```

### 3. ドキュメントの整理
- [ ] 無駄なドキュメントファイルの削除
- [ ] 既存ドキュメントの内容更新
- [ ] READMEの最新化
- [ ] API仕様書の更新
- [ ] ドキュメント命名規則への準拠確認

### 4. テストの整理
- [ ] 無駄なテストコードの削除
- [ ] テストカバレッジの確認
- [ ] **`uv run pytest` での全テスト成功確認**

## 作業手順

### Step 1: 現状分析
コードベース全体を確認し、リファクタリング対象を特定。

### Step 2: 計画作成
リファクタリング計画をユーザーに報告。

### Step 3: 実施
チェックリストに従って作業を実施。

### Step 4: テスト実行
```bash
uv run pytest -v
```
すべてのテストが成功することを確認。

### Step 5: 報告
完了した項目と結果をユーザーに報告。

## 重要なルール
- 機能変更を伴うリファクタリングは事前にユーザーに確認
- テストが通らない状態で作業を完了としない
