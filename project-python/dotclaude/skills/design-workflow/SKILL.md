# 設計ワークフロー

## 概要
このスキルは設計フェーズでの作業手順を定義します。

## ワークフロー

### Step 1: 設計書の作成
`docs` ディレクトリに以下を作成:
- 要件定義書
- 設計書（概略・機能・クラス構成）

ファイル命名規則: `YYYY-MM-DD_内容を表す日本語名.md`

### Step 2: ユーザーへの確認依頼
設計書を作成したら、必ずユーザーに確認を依頼する。

**確認依頼テンプレート:**
```
設計書を作成しました。以下の内容をご確認ください：

1. [要件定義書](docs/YYYY-MM-DD_要件定義書.md)
2. [設計書](docs/YYYY-MM-DD_設計書.md)

問題がなければ、実装フェーズに進みます。
修正が必要な点があればお知らせください。
```

### Step 3: 議論と合意形成
- 問題ないことが合意できるまでは議論を続ける
- **ユーザの承認なしに実装フェーズに進まない**

## 重要なルール
- 設計フェーズでは絶対にコードを書かない
- ユーザーの承認を得てから次のフェーズへ進む
